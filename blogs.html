Well.. were should i start ^^. I started with this project early March. 2 or 3 Weeks after the start of the Wifi course. The Wifi course was a Base programming course with C# and .netFramework. Everyone in the course had to make a project to actually pass the exam. And i said "I make a video game!". Yeah it was more than a horror trip for me.

The problem was that this kind of project was to complex for a beginner as i am to make in about 4 months something cool and actually something working to show of for a exam. So. I developt the whole game in the Unity Engine and with Visual Studio as the script editor. First i started with a simple 2D Game with some free Assets from the Unity Asset Store. 2 Weeks later i desieded to refactor the whole project from scratch and make a 3D game like Diablo with a isometric view and an abillty based combat system. Then i refactor the WHOLE Game and make a 3D game with Third Person view with an Action Orientaded combat system. And this would be the project. With this project i made it and i passed the exam. I refactored the whole project 3 times from scratch rewrote the whole logic of it and redesigend it. Kind of silly you think? Yes...

By the way the screenshot are all up to date. So. I tell you something about the logic system and the code i wrote. There is 1 Baseclass the CharacterManager who has all logic for the player and enemy. For example Health,Stamina,Attacks,Animations,Inventory, and so on. The PlayerManager and the EnemyManager derives from the CharacterManager script cause some of the logic the Player and Enemy shares and with one BaseClass i dont have to write all stuff twice. The difference between Player and EnemyManager is that the PlayerManager has Locomotion, Inputs and Inventory systems in it the Enemy dont. Cause the Enemy has no Inventory for example. The Locomotion for the Enemy and the A.I. logic is in the A.I StateMachine script. A StateMachine has several "Phases" and every phase communicates with each other. For Example: The BaseClass StateMachine: The IdleState The PursueTargetState The CombatStanceState The AttackState Every state has its logic in it and WHEN something specific happends switch to the other state and so on. For example IdleState does nothing EXCEPT. If Enemy targets player switch to PursueTargetState and so on and so on.

The Enemy StateMachine works very well and is so easy to code and understand. That i have make a desicion.. I will refactor the whole code and work more with statemachines. So i will create a new project copypaste my scripts and code in the fresh project refactor it test it and go on. I will write the whole Player logic in a statemachine the whole Enemy logic in a statemachine. Aswell as the Enviorment and Gamelevel. This is all possilbe and a so mighty feature i need to use this.

With this last words.. I have a shit whole of work to do... So... Bye i have no time at all for another blog post... :D.
